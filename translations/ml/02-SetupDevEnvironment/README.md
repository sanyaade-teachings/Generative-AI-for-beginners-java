# Java-വിനുള്ള ജനറേറ്റീവ് AI ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതി സജ്ജീകരിക്കൽ

> **ത്വരിത ആരംഭം:** Bicep + `azd` ഉപയോഗിച്ച് കുറച്ച് മിനിട്ട് കൊണ്ട് തന്നെ **Azure AI Foundry**-ൽ നിങ്ങളുടെ AI മോഡലുകൾ കോഡ്ാകയായി പ്രൊവിഷൻ ചെയ്യുക — [Azure AI Foundry സജ്ജീകരണ മാർഗ്ഗനിർദ്ദേശം](getting-started-azure-openai.md) കാണുക. ഓഥൻറിക്കേഷൻ **കീലെസ്സ്** ആണ് (Microsoft Entra ID), അതുകൊണ്ട് API കീകൾ കൈകാര്യം ചെയ്യേണ്ടതില്ല.

## നിങ്ങൾക്ക് എന്താണ് പഠിക്കേണ്ടത്

- AI അപ്ലിക്കേഷനുകൾക്കായി Java ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതി സജ്ജീകരിക്കൽ

## പരിചയം

ഈ അധ്യായം നിങ്ങളെ ഒരു ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതി സജ്ജീകരിക്കാൻ വഴികാട്ടും. ഈ കോഴ്‌സിലെ മുഴുവൻ മോഡലുകളും **Azure AI Foundry** ഉപയോഗിച്ചാണ്. Bicep കൊണ്ട് മോഡലുകൾ കോഡായി പ്രൊവിഷൻ ചെയ്യാൻ `azd` എന്ന Azure ഡെവലപ്പർ CLI ഉപയോഗിക്കുകയും, പിന്നീട് **കീലെസ്സ് ഓത്തന്റിക്കേഷൻ** (Microsoft Entra ID) ഉപയോഗിച്ച് കണക്റ്റ് ചെയ്യുകയും ചെയ്യുന്നു — API കീകൾ പകർക്കുകയോ ചോർത്തൊഴിയുകയോ വേണ്ട.

**ലോക്കൽ സജ്ജീകരണം വേണ്ട!** GitHub Codespaces ഉപയോഗിച്ച് ബ്രൗസറിലായിരിക്കും പൂർണ്ണ ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതി ലഭിക്കുക, അതിൽ നിന്നുള്ള Foundry പ്രൊവിഷൻ ചെയ്യാം.

**Azure AI Foundry** ഈ കോഴ്സിനു ഉപയോഗിക്കുന്നതിന്റെ കാരണം:
- **കോഡായി പ്രൊവിഷൻ ചെയ്യുന്നു** — ഒരു `azd up` ഓർഡറോടെ അക്കൗണ്ട്, മോഡൽ ഡിപ്ലോയ്മെന്റുകൾ പ്രൊവിഷൻ ചെയ്യുന്നു
- **കീലെസ്സ്** — Azure സൈൻ ഇൻ അല്ലെങ്കിൽ മാനേജ് ചെയ്ത ഐഡന്റിറ്റി ഉപയോഗിച്ച് ഓത്തന്റിക്കേഷൻ
- **പ്രൊഡക്ഷൻ റെഡി** — ഒരു കോഡ് ലോക്കലിലുമുമ Azure-ിലും പ്രവർത്തിക്കുന്നു
- **ലളിതമായ മാറ്റങ്ങൾ** — മോഡൽ മാറ്റണമെങ്കിൽ ഡിപ്ലോയ്മെന്റിന്റെ പേര് മാത്രം മാറും, കോഡ് മാറ്റേണ്ടതില്ല

> **ശ്രദ്ധിക്കുക**: Azure AI Foundry ഡിപ്ലോയ്മെന്റുകൾ ടോക്കൻ അടിസ്ഥാനത്തിൽ ബില്ല് ചെയ്‌തിരിക്കുന്നു (pay-as-you-go). പ്രൊവിഷൻ, റീജിയൻ, ചെലവ് വിവരങ്ങൾക്ക് [Azure AI Foundry സജ്ജീകരണ മാർഗ്ഗനിർദ്ദേശം](getting-started-azure-openai.md) കാണുക.

   > **ശ്രദ്ധിക്കുക**: അടിസ്ഥാന കോൺഫിഗ് എഡിറ്റ് ചെയ്യാൻ ആഗ്രഹിക്കുന്നുവെങ്കിൽ [Dev Container Configuration](../../../.devcontainer/devcontainer.json) കാണുക

## പടി 2: Azure AI Foundry പ്രൊവിഷൻ ചെയ്യുക

ഈ കോഴ്സിലെ AI മോഡലുകൾ Azure AI Foundry-ൽ കോഡായി പ്രൊവിഷൻ ചെയ്യുക. റിപ്പോസിറ്ററി റൂട്ട് നിന്നാണ് ഇത് ചെയ്യുക:

```bash
cd 02-SetupDevEnvironment
azd auth login
az login
azd up
```

`azd` പരിതസ്ഥിതിയുടെ പേര്, റീജിയൻ ചോദിച്ച്, `gpt-4o-mini` ഉം `text-embedding-3-small` ഉം ഉൾപ്പെടുന്ന Azure AI Foundry അക്കൗണ്ട് പ്രൊവിഷൻ ചെയ്യുന്നു, എക്സാംപിൾ `.env` ഫയലിൽ എൻഡ്പോയിന്റ് എഴുതി നൽകുന്നു — എല്ലാം കീലെസ്സ് ഓത്തന്റിക്കേഷനോടെയാണ് (API കീകൾ不要).

> **സമഗ്ര മാർഗ്ഗനിർദ്ദേശം:** മുൻപേ ആവശ്യമായ കാര്യങ്ങൾ, മാനുവൽ (പോർട്ടൽ) വഴി പ്രൊവിഷൻ, റീജിയൻ മാർഗ്ഗനിർദ്ദേശം, ചെലവ്/ക്ലീൻഅപ്പ് കുറിപ്പുകൾക്കായി [Azure AI Foundry സജ്ജീകരണ മാർഗ്ഗനിർദ്ദേശം](getting-started-azure-openai.md) കാണുക.

## പടി 3: നിങ്ങളുടെ സജ്ജീകരണം പരീക്ഷിക്കുക

നിങ്ങളുടെ Foundry മോഡലുകൾ പ്രൊവിഷൻ ചെയ്തശേഷം, [`02-SetupDevEnvironment/examples/basic-chat-azure`](../../../02-SetupDevEnvironment/examples/basic-chat-azure) ലെ ഉദാഹരണ ആപ്പ് ഉപയോഗിച്ച് കണക്ഷൻ പരീക്ഷിക്കുക.

1. നിങ്ങളുടെ ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതിയിൽ ടെർമിനൽ തുറക്കുക.
2. ഉദാഹരണ ഫോൾഡറിൽ നീങ്ങുക:
   ```bash
   cd 02-SetupDevEnvironment/examples/basic-chat-azure
   ```
3. ഉറപ്പാക്കുക നിങ്ങൾ സൈൻ ഇൻ ചെയ്‌തിരിക്കുന്നു (കീലെസ്സ് ഓത്തൻറിക്കേഷനു ടോക്കൺ ആവശ്യമുണ്ട്):
   ```bash
   az login
   ```
   > നിങ്ങൾ `azd up` ഓടിച്ചിട്ടുണ്ടെങ്കിൽ `.env` ഫയലിൽ എൻഡ്പോയിന്റ് ഇതിനകം എഴുതി വച്ചിരിക്കുന്നതാണ്.
4. ആപ്പ് റൺ ചെയ്യുക:
   ```bash
   mvn clean spring-boot:run
   ```

`gpt-4o-mini` മോഡലിൽ നിന്ന് ഒരു പ്രതികരണം നിങ്ങൾക്ക് കാണാം.

### ഉദാഹരണ കോഡ് മനസിലാക്കുക

`examples/basic-chat-azure` എന്ന ഫോള्डറിൽ ഉള്ള ഉദാഹരണ കോഡ് Spring Boot ആപ്പ് ആണ്, ഇത് **Spring AI** ഉപയോഗിച്ച് Azure AI Foundry-യുമായി കീലെസ്സ് ഓത്തൻറിക്കേഷൻ ഉപയോഗിച്ച് കണക്റ്റ് ചെയ്യുന്നു.

**ഈ കോഡ് ചെയ്യുന്നത്:**
- Azure AI Foundry-യുമായി നിങ്ങളുടെ Azure സൈൻ ഇൻ (Microsoft Entra ID) വഴി കണക്റ്റ് ചെയ്യുന്നു — API കീ不要
- `gpt-4o-mini` മോഡലിലേക്ക് പ്രോംപ്റ്റ് അയയ്ക്കുന്നു
- AI യുടെ പ്രതികരണം സ്വീകരിക്കുകയും പ്രദർശിപ്പിക്കുകയും ചെയ്യുന്നു
- നിങ്ങളുടെ സജ്ജീകരണം ശരിയാണെന്ന് സ്ഥിരീകരിക്കുന്നു

**പ്രധാന ഡിപ്പൻഡൻസി** (`pom.xml`):
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-azure-openai</artifactId>
</dependency>
```

**കോൺഫിഗറേഷൻ** (`application.yml`):
```yaml
spring:
  ai:
    azure:
      openai:
        # Endpoint only - no api-key. Spring AI uses DefaultAzureCredential (keyless).
        endpoint: ${AZURE_OPENAI_ENDPOINT}
        chat:
          options:
            deployment-name: ${AZURE_OPENAI_DEPLOYMENT:gpt-4o-mini}
```

നന്നായി! ഇപ്പോൾ നിങ്ങൾക്കുണ്ട്:

- Bicep + `azd` ഉപയോഗിച്ച് Azure AI Foundry മോഡലുകൾ കോഡായി പ്രൊവിഷൻ ചെയ്തത്
- Java ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതി പ്രവർത്തനത്തിൽ (Codespaces, dev containers, അല്ലെങ്കിൽ ലോക്കൽ)
- Azure AI Foundry-യുമായി കീലെസ്സ് ഓത്തൻറിക്കേഷൻ (Microsoft Entra ID) ഉപയോഗിച്ച് കണക്റ്റ് ചെയ്തത് — API കീ不要
- ഏറ്റവും ലളിതമായ ഉദാഹരണത്തിൽ നമ്മുടെ മോഡലുമായി സംവദിച്ച് എല്ലാം ശരിയെന്ന് പരിശോധന ചെയ്തത്

## സങ്കീർണ്ണതകൾ പരിഹരിക്കൽ

പ്രശ്‌നം ഉണ്ടാകുമോ? സാധാരണ പ്രശ്‌നങ്ങളും പരിഹാരങ്ങളും:

- **ഓത്തന്റിക്കേഷൻ പരാജയപ്പെട്ടു (401/403)?** 
  - `az login` റൺ ചെയ്‌തു സൈൻ ഇൻ ആണെങ്കിൽ ഉറപ്പാക്കുക — ഓത്തൻറിക്കേഷൻ കീലെസ്സ് ആണ്
  - നിങ്ങളുടെ അക്കൗണ്ടിന് റിസോഴ്സിൽ **Cognitive Services OpenAI User** റോളുണ്ട് എന്ന് ഉറപ്പാക്കുക
  - അടുത്തിടെ പ്രൊവിഷൻ ചെയ്തുവെങ്കിൽ, റോളിന്റെ പ്രചരണം ഒരു മിനിറ്റ് വരെ കാത്തിരിക്കുക

- **Maven കണ്ടെത്താനായിട്ടില്ല?** 
  - dev containers/Codespaces ഉപയോഗിച്ചാൽ Maven മുൻകൂട്ടി ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ട്
  - ലോക്കൽ സജ്ജീകരണത്തിനായി Java 21+യും Maven 3.9+യും ഇൻസ്റ്റാൾ ചെയ്യപ്പെട്ടെന്ന് ഉറപ്പാക്കുക
  - ഇൻസ്റ്റലേഷൻ പരിശോധിക്കാൻ `mvn --version` പരീക്ഷിക്കുക

- **`azd` കണ്ടെത്താനായില്ല അല്ലെങ്കിൽ പ്രൊവിഷൻ പരാജയപ്പെട്ടു?** 
  - [Azure Developer CLI](https://aka.ms/azure-dev/install) ഇൻസ്റ്റാൾ ചെയ്ത് `azd auth login` നടത്തുക
  - `gpt-4o-mini` ലഭ്യമായ റീജിയൻ തിരഞ്ഞെടുക്കുക (ഉദാ: `eastus2`)
  - വിശദാംശങ്ങൾക്ക് [Azure AI Foundry സജ്ജീകരണം](getting-started-azure-openai.md) കാണുക

- **Dev കണ്ടയ്നർ ആരംഭിക്കുന്നില്ല?** 
  - Docker Desktop പ്രവർത്തനത്തിലാണ് എന്ന് നോക്കുക (ലോക്കൽ ഡെവലപ്പ്മെന്റിന്)
  - കണ്ടയ്നർ വീണ്ടും നിർമ്മിക്കാൻ ശ്രമിക്കുക: `Ctrl+Shift+P` → "Dev Containers: Rebuild Container"

- **അപ്പ്ലിക്കേഷൻ കമ്പൈൽ ചെയ്യുമ്പോൾ പിശകുകൾ?**
  - ശരിയായ ഡയറക്ടറിയിൽ തന്നെയെന്ന് ഉറപ്പാക്കുക: `02-SetupDevEnvironment/examples/basic-chat-azure`
  - ക്ലീൻ ചെയ്ത് വീണ്ടും കമ്പൈൽ ചെയ്യാൻ ശ്രമിക്കുക: `mvn clean compile`

> **സഹായം വേണോ?** ഇപ്പോഴും പ്രശ്‌നമുണ്ടോ? റിപ്പോസിറ്ററിയിൽ ഒരു issue തുറക്കൂ, ഞങ്ങൾ സഹായിക്കും.
