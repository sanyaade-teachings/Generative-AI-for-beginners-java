# AGENTS.md

## പ്രോജക്ട് ഓവർവ്യൂ

Java ഉപയോഗിച്ച് Generative AI വികസനം പഠിക്കുന്നതിനുള്ള ഈ ഒരു വിദ്യാഭ്യാസ റിപ്പോസിറ്ററിയാണ്. ഇത് വലിയ ഭാഷാ മോഡലുകൾ (LLMs), പ്രോംപ്റ്റ് എഞ്ചിനീയറിങ്, എൻബെഡിങ്ങുകൾ, RAG (Retrieval-Augmented Generation), കൂടാതെ മോഡൽ കോൺടെക്സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) എന്നിവയെ ഉൾക്കൊള്ളിക്കുന്ന സമഗ്രമായ ഹാൻഡ്‌സ്-ഓൺ കോഴ്സാണ് നൽകുന്നത്.

**പ്രധാന സാങ്കേതികവിദ്യകൾ:**
- Java 21
- Spring Boot 3.5.x
- Spring AI 1.1.x
- Maven
- LangChain4j
- Azure AI Foundry, Azure OpenAI, OpenAI SDKകൾ

**ആർക്കിടെക്ചർ:**
- അദ്ധ്യായങ്ങളായി വ്യവസ്ഥാപിതമായ പെട്ടെന്ന് പ്രവർത്തിക്കുന്ന Spring Boot അപ്ലിക്കേഷനുകളുടെ ബഹുലത
- വ്യത്യസ്ത AI മാതൃകകൾ പ്രദർശിപ്പിക്കുന്ന സാമ്പിൾ പ്രോജക്റ്റുകൾ
- മുൻകൂട്ടി ക്രമീകരിച്ച dev കൺടെയിനറുകൾ ഉൾപ്പെടുത്തി GitHub Codespaces-ന്റെ ക്രമീകരണം

### കോൺഫിഗറേഷൻ

**Azure AI Foundry സെറ്റപ്പ് (കീസ് ഇല്ലാത്ത, ശിപാർശ ചെയ്യുന്നു):**
```bash
# ഫൗണ്ട്രി അക്കൗണ്ടും മോഡൽ ഡിപ്പ്ലോയ്മെന്റും കോഡായി ഒരുക്കുക
cd 02-SetupDevEnvironment
azd auth login
az login
azd up
# azd നിങ്ങളുടെ എൻഡ്‌പോയിന്റ് ഉപയോഗിച്ച് examples/basic-chat-azure/.env എഴുതുന്നു (കീ ഇല്ലാതെ)
```

**മാനുവൽ എണ്ട്‌പോയിന്റ് കോൺഫിഗ്:**
```bash
# നിങ്ങൾ azd ഉപയോഗിക്കാത്ത പക്ഷം, എൻഡ്‌പോയിന്റ് സ്വയം സജ്ജമാക്കുക (auth az login വഴി കീലെസ് ആയി തുടരുന്നു)
cd 02-SetupDevEnvironment/examples/basic-chat-azure
cp .env.example .env
# .env എഡിറ്റ് ചെയ്യുക: AZURE_OPENAI_ENDPOINT=https://<resource>.openai.azure.com/
```

### അപ്ലിക്കേഷനുകൾ പ്രവർത്തിപ്പിക്കൽ

**Spring Boot അപ്ലിക്കേഷൻ പ്രവർത്തിപ്പിക്കുക:**
```bash
cd [project-directory]
mvn spring-boot:run
```

**ഒരു പ്രോജക്റ്റ് ബിൽഡ് ചെയ്യുക:**
```bash
cd [project-directory]
mvn clean install
```

**MCP കാൽക്കുലേറ്റർ സെർവർ ആരംഭിക്കുക:**
```bash
cd 04-PracticalSamples/calculator
mvn spring-boot:run
# സെർവർ http://localhost:8080 ൽ പ്രവർത്തിക്കുന്നു
```

**ക്ലയന്റ് ഉദാഹരണങ്ങൾ പ്രവർത്തിപ്പിക്കുക:**
```bash
# മറ്റൊരു ടെർമിനലിൽ സെർവർ ആരംഭിച്ചതിനു ശേഷം
cd 04-PracticalSamples/calculator

# നേരിട്ടുള്ള MCP ക്ലൈയന്റ്
mvn exec:java -Dexec.mainClass="com.microsoft.mcp.sample.client.SDKClient"

# AI-ച്ചാലीत ക്ലൈയന്റ് (AZURE_OPENAI_ENDPOINT + az login ആവശ്യമാണ്)
mvn exec:java -Dexec.mainClass="com.microsoft.mcp.sample.client.LangChain4jClient"

# ഇന്ററാക്ടീവ് ബോട്ട്
mvn exec:java -Dexec.mainClass="com.microsoft.mcp.sample.client.Bot"
```

- കാൽക്കുലേറ്റർ പ്രോജക്റ്റിലുള്ള ക്ലയന്റ് ഉദാഹരണങ്ങൾ `src/test/java/com/microsoft/mcp/sample/client/`-ൽ ഉണ്ട്

### മാനുവൽ ടെസ്റ്റിംഗ്
എല്ലാമില്ലാതെ കൈകാര്യം ചെയ്യേണ്ട അനേകം ഉദാഹരണങ്ങൾ ഇന്ററാക്ടീവ് അപ്ലിക്കേഷനുകളാണ്:

1. `mvn spring-boot:run` ഉപയോഗിച്ച് അപ്ലിക്കേഷൻ ആരംഭിക്കുക
2. എൻഡ്പോയിന്റുകൾ പരീക്ഷിക്കുക അല്ലെങ്കിൽ CLI-യുമായി ഇടപഴകുക
3. ഓരോ പ്രോജക്റ്റിന്റെ README.md ലേഖനത്തിലുള്ള രേഖാസഞ്ചയം അനുസരിച്ച് പ്രതീക്ഷിച്ച പ്രవర്ത്തനം സ്ഥിരീകരിക്കുക

## കോഡ് സ്റ്റൈൽ മാർഗ്ഗനിർദ്ദേശങ്ങൾ

### Java നിശ്ചിതാനുസൃതികളും
- **ജാവ വേർഷൻ:** Java 21 മോടേൺ സവിശേഷതകളോടെ
- **സ്റ്റൈൽ:** പ്രമാണിക്കപ്പെട്ട ജാവ മുഖ്യരീതികൾ പാലിക്കുക
- **പേര് നൽകിയിട്ടുള്ള രീതി:**
  - ക്ലാസുകൾ: PascalCase
  - മേത്തഡുകളും വേരിയബിൾസ്: camelCase
  - സ്ഥിരം മൂല്യങ്ങൾ: UPPER_SNAKE_CASE
  - പാക്കേജ് നാമങ്ങൾ: ചെറിയ അക്ഷരങ്ങൾ

### Spring Boot മാതൃകകൾ
- ബിസിനസ് ലജിക് @Service ഉപയോഗിക്കുക
- REST എൻഡ്പോയ്‌ന്റ്റുകൾക്ക് @RestController ഉപയോഗിക്കുക
- കോൺഫിഗറേഷൻ application.yml അല്ലെങ്കിൽ application.properties വഴി
- हार্ড്കോഡ് മൂല്യങ്ങൾക്കുപകരം പാരിസ്ഥിതിക വ്യത്യാസങ്ങൾ മുൻഗണിക്കുക
- MCP പരസ്യപ്പെടുത്തിയ മേത്തഡുകൾക്ക് @Tool അനുബന്ധിക്കുക

### Azure AI Foundry

- **കോഡായി പ്രൊവിഷൻ ചെയ്തിട്ടുണ്ട്:** Bicep + azd (`azd up`) അക്കൗണ്ടും മോഡൽ ഡിപ്ലോയ്മെന്റുകളും സൃഷ്ടിക്കുന്നു

### ബഹുസംഖ്യ പ്രോജക്റ്റുകളുമായി ജോലി ചെയ്യൽ
ഓരോ സാമ്പിൾ പ്രോജക്റ്റും സ്വതന്ത്രമായി പ്രവർത്തിക്കുന്ന വിധത്തിലാണ്:
```bash
# പ്രത്യേക പ്രോജക്ടിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക
cd 04-PracticalSamples/[project-name]

# ഓരോതിലും തങ്ങളുടെ സ്വന്തം pom.xml ഉണ്ട്, സ്വതന്ത്രമായി നിർമ്മിക്കാവുന്നതാണ്
mvn clean install
```

### സാധാരണ പ്രശ്നങ്ങൾ

**Java വേർഷൻ പൊരുത്തക്കേട്:**
```bash
# ജാവ 21 സ്ഥിരീകരിക്കുക
java -version
# ആവശ്യമെങ്കിൽ JAVA_HOME അപ്ഡേറ്റ് ചെയ്യുക
export JAVA_HOME=/usr/lib/jvm/msopenjdk-current
```

**ഡിപെൻഡൻസി ഡൗൺലോഡ് പ്രശ്നങ്ങൾ:**
```bash
# Maven കാഷെ പരസിർത്തി വീണ്ടും ശ്രമിക്കുക
rm -rf ~/.m2/repository
mvn clean install
```

**എൻഡ്പോയിന്റ് അല്ലെങ്കിൽ സൈൻ ഇൻ കണ്ടെത്താനായില്ല:**
```bash
# നിലവിലെ സെഷനിൽ എൻഡ്‌പോയിന്റെ് സജ്ജമാക്കുക കൂടാതെ സൈൻ ഇൻ ചെയ്യുക (കീലെസ്)
export AZURE_OPENAI_ENDPOINT="https://your-resource.openai.azure.com/"
az login

# അല്ലെങ്കിൽ പ്രോജക്ട് ഡയറക്ടറിയിലുള്ള .env ഫയൽ ഉപയോഗിക്കുക
echo "AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/" > .env
```

**പോർട്ട് ഇതിനകം ഉപയോഗത്തിലാണ്:**
```bash
# ഡീഫാൾട്ടായി സ്പ്രിംഗ് ബൂട്ട് 8080 പോർട്ട് ഉപയോഗിക്കുന്നു
# application.properties-ൽ മാറ്റം:
server.port=8081
```
