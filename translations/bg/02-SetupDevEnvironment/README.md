# Настройване на среди за разработка за Generative AI за Java

> **Бърз старт:** Осигурете вашите AI модели на **Azure AI Foundry** като код с Bicep + `azd` за няколко минути — вижте [Ръководство за настройка на Azure AI Foundry](getting-started-azure-openai.md). Удостоверяването е **без ключ** (Microsoft Entra ID), така че няма API ключове за управление.

## Какво ще научите

- Настройване на среда за разработка на Java за AI приложения
- Избор и конфигуриране на предпочитаната среда за разработка (първо в облака с Codespaces, локален контейнер за разработка или пълна локална настройка)
- Тестване на настройката чрез свързване към модел на Azure AI Foundry

## Въведение

Тази глава ще ви преведе през настройването на среда за разработка. Ще използваме **Azure AI Foundry** за моделите през целия курс. Вие осигурявате моделите като код с Bicep и Azure Developer CLI (`azd`), след което се свързвате с **удостоверяване без ключ** (Microsoft Entra ID) — без API ключове за копиране или изтичане.

**Не е необходима локална настройка!** Можете да използвате GitHub Codespaces, който предоставя пълна среда за разработка в браузъра ви, и да осигурите Foundry оттам.

Използваме **Azure AI Foundry** за този курс, защото то е:
- **Осигурено като код** — една команда `azd up` разгръща акаунта и разгръщанията на моделите
- **Без ключ** — удостоверяване със вашия Azure вход или управлявана идентичност
- **Готово за продукция** — същият код работи локално и в Azure
- **Гъвкаво** — сменяте моделите чрез промяна на име на разгръщане, без да променяте кода си

> **Забележка**: Разгръщанията на Azure AI Foundry се таксуват на база токени (плащаш според употребата). Вижте [Ръководството за настройка на Azure AI Foundry](getting-started-azure-openai.md) за подробности по осигуряването, регион и цени.


## Стъпка 1: Настройване на среда за разработка

<a name="quick-start-cloud"></a>

Създадохме предварително конфигуриран контейнер за разработка, за да минимизираме времето за настройка и да гарантираме, че разполагате с всички необходими инструменти за този курс Generative AI за Java. Изберете предпочитания от вас подход за разработка:

### Опции за настройка на средата:

#### Опция А: GitHub Codespaces (Препоръчано)

**Започнете да пишете код за 2 минути - не е нужна локална настройка!**

1. Форкнете това хранилище във вашия GitHub акаунт
   > **Забележка**: Ако искате да редактирате основната конфигурация, моля разгледайте [Конфигурация на контейнера за разработка](../../../.devcontainer/devcontainer.json)
2. Натиснете **Code** → таб **Codespaces** → **...** → **New with options...**
3. Използвайте по подразбиране – това ще селектира **Dev container configuration**: **Generative AI Java Development Environment** персонализиран devcontainer, създаден за този курс
4. Натиснете **Create codespace**
5. Изчакайте около 2 минути средата да се зареди
6. Продължете към [Стъпка 2: Осигурете Azure AI Foundry](#стъпка-2-осигуряване-на-azure-ai-foundry)

#### Опция Б: Локален контейнер за разработка

**За разработчици, предпочитащи локална разработка с Docker**

1. Форкнете и клонирайте това хранилище на локалната си машина
   > **Забележка**: Ако искате да редактирате основната конфигурация, моля разгледайте [Конфигурация на контейнера за разработка](../../../.devcontainer/devcontainer.json)

4. Отворете папката на хранилището във VS Code
5. Когато получите подкана, натиснете **Reopen in Container** (или използвайте `Ctrl+Shift+P` → "Dev Containers: Reopen in Container")
6. Изчакайте контейнера да се построи и стартира
7. Продължете към [Стъпка 2: Осигуряване на Azure AI Foundry](#стъпка-2-осигуряване-на-azure-ai-foundry)

## Стъпка 2: Осигуряване на Azure AI Foundry

Разгърнете AI моделите от курса в Azure AI Foundry като код. От корена на хранилището:

```bash
cd 02-SetupDevEnvironment
azd auth login
az login
azd up
```

`azd` ви пита за име на среда и регион, осигурява акаунт в Azure AI Foundry с разгръщания `gpt-4o-mini` и `text-embedding-3-small` и записва крайна точка в примерния `.env` — всичко с **удостоверяване без ключ** (без API ключове).

> **Пълно ръководство:** Вижте [Ръководството за настройка на Azure AI Foundry](getting-started-azure-openai.md) за предварителни условия, алтернатива чрез портала, насоки за регионите и бележки за цена/почистване.

## Стъпка 3: Тестване на настройката

След като вашите модели Foundry са осигурени, тествайте връзката с примерното приложение в [`02-SetupDevEnvironment/examples/basic-chat-azure`](../../../02-SetupDevEnvironment/examples/basic-chat-azure).

1. Отворете терминала във вашата среда за разработка.
2. Отидете до примера:
   ```bash
   cd 02-SetupDevEnvironment/examples/basic-chat-azure
   ```
3. Уверете се, че сте влезли (удостоверяване без ключ се нуждае от токен):
   ```bash
   az login
   ```
   > Ако сте стартирали `azd up`, файлът `.env` с вашата крайна точка вече е записан вместо вас.
4. Стартирайте приложението:
   ```bash
   mvn clean spring-boot:run
   ```

Трябва да видите отговор от модела `gpt-4o-mini`.

### Разбиране на примерния код

Примерът в `examples/basic-chat-azure` е Spring Boot приложение, което използва **Spring AI** за връзка с Azure AI Foundry с удостоверяване без ключ.

**Какво прави този код:**
- **Свързва се** с Azure AI Foundry чрез вашия Azure вход (Microsoft Entra ID) — без API ключ
- **Изпраща** подканa към модела `gpt-4o-mini`
- **Получава** и показва отговора на AI
- **Проверява** дали вашата настройка работи коректно

**Ключова зависимост** (в `pom.xml`):
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-azure-openai</artifactId>
</dependency>
```

**Конфигурация** (`application.yml`):
```yaml
spring:
  ai:
    azure:
      openai:
        # Endpoint only - no api-key. Spring AI uses DefaultAzureCredential (keyless).
        endpoint: ${AZURE_OPENAI_ENDPOINT}
        chat:
          options:
            deployment-name: ${AZURE_OPENAI_DEPLOYMENT:gpt-4o-mini}
```

## Отстраняване на проблеми

Имате проблеми? Ето някои често срещани проблеми и решения:

- **Неуспешно удостоверяване (401/403)?** 
  - Стартирайте `az login` — удостоверяването е без ключове, трябва да сте влезли
  - Проверете дали вашият акаунт има ролята **Cognitive Services OpenAI User** върху ресурса
  - Ако току-що сте осигурили, изчакайте минута за разпространение на ролите

- **Не намира Maven?** 
  - Ако използвате dev контейнер/ Codespaces, Maven трябва да е предварително инсталиран
  - За локална настройка, уверете се, че имате Java 21+ и Maven 3.9+ инсталирани
  - Опитайте `mvn --version` за проверка на инсталацията

- **`azd` не се намира или осигуряването неуспешно?** 
  - Инсталирайте [Azure Developer CLI](https://aka.ms/azure-dev/install) и стартирайте `azd auth login`
  - Изберете регион, където е наличен `gpt-4o-mini` (например `eastus2`)
  - Вижте [Ръководството за настройка на Azure AI Foundry](getting-started-azure-openai.md) за подробности

- **Dev контейнерът не се стартира?** 
  - Уверете се, че Docker Desktop работи (за локална разработка)
  - Опитайте да преизградите контейнера: `Ctrl+Shift+P` → "Dev Containers: Rebuild Container"

- **Грешки при компилация на приложението?**
  - Уверете се, че сте в правилната директория: `02-SetupDevEnvironment/examples/basic-chat-azure`
  - Опитайте да почистите и преизградите: `mvn clean compile`

> **Нуждаете се от помощ?**: Все още имате проблеми? Отворете issue в хранилището и ще ви помогнем.
