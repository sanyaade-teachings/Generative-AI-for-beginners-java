# إعداد بيئة التطوير للذكاء الاصطناعي التوليدي لجافا

> **بدء سريع:** قم بتوفير نماذج الذكاء الاصطناعي الخاصة بك على **Azure AI Foundry** ككود باستخدام Bicep + `azd` خلال دقائق معدودة — اطلع على [دليل إعداد Azure AI Foundry](getting-started-azure-openai.md). المصادقة **بدون مفاتيح** (Microsoft Entra ID)، لذا لا يوجد مفاتيح API لإدارتها.

## ما ستتعلمه

- إعداد بيئة تطوير جافا لتطبيقات الذكاء الاصطناعي
- اختيار وتكوين بيئة التطوير المفضلة لديك (السحابة أولاً مع Codespaces، حاوية تطوير محلية، أو إعداد محلي كامل)
- اختبار إعدادك من خلال الاتصال بنموذج Azure AI Foundry

## مقدمة

سيُرشدك هذا الفصل خلال إعداد بيئة تطوير. سوف نستخدم **Azure AI Foundry** للنماذج طوال هذا المساق. تقوم بتوفير النماذج ككود باستخدام Bicep و Azure Developer CLI (`azd`)، ثم تتصل بمصادقة **بدون مفتاح** (Microsoft Entra ID) — لا نسخ أو تسريب لمفاتيح API.

**لا حاجة لإعداد محلي!** يمكنك استخدام GitHub Codespaces، الذي يوفر بيئة تطوير كاملة في متصفحك، وتوفير Foundry من هناك.

نستخدم **Azure AI Foundry** لهذا المساق لأنه:
- **يتم توفيره ككود** — أمر واحد `azd up` ينشر الحساب ونشر النماذج
- **بدون مفتاح** — المصادقة عبر تسجيل الدخول إلى Azure الخاص بك أو هوية مُدارة
- **جاهز للإنتاج** — نفس الكود يعمل محليًا وفي Azure
- **مرن** — استبدل النماذج بتغيير اسم النشر، وليس الكود الخاص بك

> **ملاحظة**: يتم احتساب تكاليف نشر Azure AI Foundry حسب عدد الرموز (ادفع حسب الاستخدام). راجع [دليل إعداد Azure AI Foundry](getting-started-azure-openai.md) للحصول على تفاصيل التوفير والمنطقة والتكلفة.

## الخطوة 1: إعداد بيئة التطوير الخاصة بك

<a name="quick-start-cloud"></a>

لقد أنشأنا حاوية تطوير مُهيأة مسبقًا لتقليل وقت الإعداد وضمان وجود كل الأدوات اللازمة لهذا المساق في الذكاء الاصطناعي التوليدي لجافا. اختر نهج التطوير المفضل لديك:

### خيارات إعداد البيئة:

#### الخيار أ: GitHub Codespaces (موصى به)

**ابدأ البرمجة في دقيقتين - لا حاجة لإعداد محلي!**

1. قم بعمل فورك لهذا المستودع إلى حساب GitHub الخاص بك  
   > **ملاحظة**: إذا كنت تريد تعديل الإعدادات الأساسية يرجى الاطلاع على [تكوين حاوية التطوير](../../../.devcontainer/devcontainer.json)
2. انقر على **Code** → علامة تبويب **Codespaces** → **...** → **جديد مع الخيارات...**
3. استخدم الإعدادات الافتراضية – سيتم اختيار **تكوين حاوية التطوير**: **بيئة تطوير الذكاء الاصطناعي التوليدي لجافا** التي تم إنشاؤها خصيصًا لهذا المساق
4. انقر على **إنشاء الكودسبيس**
5. انتظر حوالي دقيقتين حتى تصبح البيئة جاهزة
6. انتقل إلى [الخطوة 2: توفير Azure AI Foundry](#الخطوة-2-توفير-azure-ai-foundry)

**للمطورين الذين يفضلون التطوير المحلي باستخدام Docker**

6. انتظر حتى يتم بناء وتشغيل الحاوية
7. انتقل إلى [الخطوة 2: توفير Azure AI Foundry](#الخطوة-2-توفير-azure-ai-foundry)

## الخطوة 2: توفير Azure AI Foundry

قم بنشر نماذج الذكاء الاصطناعي الخاصة بالمساق إلى Azure AI Foundry ككود. من جذر المستودع:

```bash
cd 02-SetupDevEnvironment
azd auth login
az login
azd up
```
  
يقوم `azd` بطلب اسم البيئة والمنطقة، ويوفر حساب Azure AI Foundry مع نشرات `gpt-4o-mini` و `text-embedding-3-small`، ويكتب نقطة النهاية في ملف `.env` الخاص بالمثال — كل ذلك مع **مصادقة بدون مفتاح** (لا مفاتيح API).

> **شرح كامل:** راجع [دليل إعداد Azure AI Foundry](getting-started-azure-openai.md) للمتطلبات المسبقة، البديل اليدوي (المدخل)، إرشادات المناطق، وملاحظات التكلفة والتنظيف.

## الخطوة 3: اختبار الإعداد الخاص بك

بعد توفير نماذج Foundry الخاصة بك، اختبر الاتصال مع تطبيق المثال في [`02-SetupDevEnvironment/examples/basic-chat-azure`](../../../02-SetupDevEnvironment/examples/basic-chat-azure).

1. افتح الطرفية في بيئة التطوير الخاصة بك.  
2. انتقل إلى المثال:  
   ```bash
   cd 02-SetupDevEnvironment/examples/basic-chat-azure
   ```
  
3. تأكد من أنك مسجل الدخول (المصادقة بدون مفتاح تحتاج إلى رمز مميز):  
   ```bash
   az login
   ```
  
> إذا قمت بتشغيل `azd up`، فقد تم كتابة ملف `.env` مع نقطة النهاية تلقائيًا.  
4. شغّل التطبيق:  
   ```bash
   mvn clean spring-boot:run
   ```
  
يجب أن ترى استجابة من نموذج `gpt-4o-mini`.

### فهم كود المثال

المثال تحت `examples/basic-chat-azure` هو تطبيق Spring Boot يستخدم **Spring AI** للاتصال بـ Azure AI Foundry بمصادقة بدون مفتاح.

**ما يفعله هذا الكود:**  
- **يتصل** بـ Azure AI Foundry باستخدام تسجيل الدخول الخاص بك في Azure (Microsoft Entra ID) — بدون مفتاح API  
- **يرسل** عبارة للنموذج `gpt-4o-mini`  
- **يتلقى** ويعرض استجابة الذكاء الاصطناعي  
- **يتحقق** من عمل الإعداد بشكل صحيح

**التبعية الرئيسية** (في `pom.xml`):  
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-azure-openai</artifactId>
</dependency>
```
  
**التكوين** (`application.yml`):  
```yaml
spring:
  ai:
    azure:
      openai:
        # Endpoint only - no api-key. Spring AI uses DefaultAzureCredential (keyless).
        endpoint: ${AZURE_OPENAI_ENDPOINT}
        chat:
          options:
            deployment-name: ${AZURE_OPENAI_DEPLOYMENT:gpt-4o-mini}
```


## الملخص

رائع! لديك الآن كل شيء مُعد:

- نماذج Azure AI Foundry تم توفيرها ككود مع Bicep + `azd`  
- تشغيل بيئة تطوير جافا (سواء Codespaces، حاويات التطوير، أو محليًا)  
- الاتصال بـ Azure AI Foundry بمصادقة بدون مفتاح (Microsoft Entra ID) — لا مفاتيح API  
- اختبار كل شيء يعمل مع مثال بسيط يتحدث إلى نموذجك

## استكشاف الأخطاء وإصلاحها

هل تواجه مشاكل؟ إليك المشاكل الشائعة وحلولها:

- **فشل المصادقة (401/403)؟**  
  - شغّل `az login` — المصادقة بدون مفتاح، لذا يجب أن تكون مسجل الدخول  
  - تحقق من أن حسابك لديه دور **Cognitive Services OpenAI User** على المورد  
  - إذا قمت للتو بالتوفير، انتظر دقيقة لتعميم تعيين الدور  

- **Maven غير موجود؟**  
  - إذا كنت تستخدم حاويات التطوير/Codespaces، يجب أن يكون Maven مثبّتًا مسبقًا  
  - للإعداد المحلي، تأكد من تثبيت Java 21+ و Maven 3.9+  
  - جرب `mvn --version` للتحقق  

- **`azd` غير موجود أو يفشل التوفير؟**  
  - ثبّت [Azure Developer CLI](https://aka.ms/azure-dev/install) وشغّل `azd auth login`  
  - اختر منطقة يتوفر فيها `gpt-4o-mini` (مثلاً `eastus2`)  
  - راجع [دليل إعداد Azure AI Foundry](getting-started-azure-openai.md) للمزيد  

- **حاوية التطوير لا تبدأ؟**  
  - تأكد من تشغيل Docker Desktop (للتطوير المحلي)  
  - حاول إعادة بناء الحاوية: `Ctrl+Shift+P` → "Dev Containers: Rebuild Container"  

- **أخطاء تجميع التطبيق؟**  
  - تأكد من أنك في الدليل الصحيح: `02-SetupDevEnvironment/examples/basic-chat-azure`  
  - جرب تنظيف المشروع وإعادة البناء: `mvn clean compile`  

> **تحتاج مساعدة؟**: ما زلت تواجه مشاكل؟ افتح قضية في المستودع وسنساعدك.
