# การใช้งานจริงและโครงการ

## สิ่งที่คุณจะได้เรียนรู้
ในส่วนนี้เราจะแสดงตัวอย่างสามแอปพลิเคชันที่ใช้งานจริงซึ่งแสดงรูปแบบการพัฒนา AI สร้างสรรค์ด้วย Java:
- สร้างเครื่องกำเนิดเรื่องสัตว์เลี้ยงมัลติ-โหมดที่รวม AI ฝั่งไคลเอนต์และฝั่งเซิร์ฟเวอร์
- นำ AI โมเดลภายในเครื่องมาใช้กับการสาธิต Foundry Local Spring Boot
- พัฒนาบริการ Model Context Protocol (MCP) ด้วยตัวอย่างเครื่องคิดเลข

## บทนำ

บทนี้แสดง **โครงการตัวอย่าง** ที่สาธิตรูปแบบการพัฒนา AI สร้างสรรค์ด้วย Java แต่ละโครงการทำงานได้เต็มที่และแสดงเทคโนโลยี AI รูปแบบสถาปัตยกรรม และแนวปฏิบัติที่ดีที่สุดที่คุณสามารถปรับใช้กับแอปของคุณเองได้

### การสาธิต Foundry Local Spring Boot

**[การสาธิต Foundry Local Spring Boot](foundrylocal/README.md)** แสดงวิธีการรวมกับโมเดล AI ภายในเครื่องโดยใช้ **OpenAI Java SDK** สาธิตการเชื่อมต่อกับโมเดลที่รันบน Foundry Local (เช่น **Phi-4-mini**) พร้อมการตรวจจับโมเดลอัตโนมัติ ช่วยให้คุณสามารถรันแอป AI โดยไม่ต้องพึ่งบริการคลาวด์

### เครื่องกำเนิดเรื่องสัตว์เลี้ยง

**[เครื่องกำเนิดเรื่องสัตว์เลี้ยง](petstory/README.md)** คือแอปเว็บ Spring Boot ที่น่าสนใจซึ่งสาธิต **การประมวลผล AI มัลติ-โหมด** เพื่อสร้างเรื่องราวสัตว์เลี้ยงแบบสร้างสรรค์ มันรวมความสามารถ AI ฝั่งไคลเอนต์และฝั่งเซิร์ฟเวอร์ โดยใช้ transformer.js สำหรับโต้ตอบ AI ในเบราว์เซอร์และ OpenAI SDK สำหรับประมวลผลฝั่งเซิร์ฟเวอร์

### บริการเครื่องคิดเลข MCP (การสาธิต MCP สำหรับผู้เริ่มต้น)

**[บริการเครื่องคิดเลข MCP](calculator/README.md)** เป็นการสาธิตง่าย ๆ ของ **Model Context Protocol (MCP)** โดยใช้ Spring AI ให้ความรู้เบื้องต้นที่เหมาะสำหรับผู้เริ่มต้นเกี่ยวกับแนวคิด MCP โดยแสดงวิธีสร้าง MCP Server พื้นฐานที่โต้ตอบกับ MCP Clients ได้

## ลำดับการเรียนรู้

โครงการเหล่านี้ออกแบบมาเพื่อต่อยอดแนวคิดจากบทก่อนหน้า:

1. **เริ่มต้นง่าย ๆ**: เริ่มต้นด้วยการสาธิต Foundry Local Spring Boot เพื่อเข้าใจการรวม AI กับโมเดลภายในเครื่องขั้นพื้นฐาน
2. **เพิ่มความโต้ตอบ**: ก้าวต่อไปยังเครื่องกำเนิดเรื่องสัตว์เลี้ยงเพื่อเรียนรู้ AI มัลติ-โหมดและการโต้ตอบผ่านเว็บ
3. **เรียนรู้พื้นฐาน MCP**: ทดลองบริการเครื่องคิดเลข MCP เพื่อเข้าใจพื้นฐานของ Model Context Protocol

## สรุป

ทำได้ดีมาก! คุณได้สำรวจแอปพลิเคชันจริงบางอย่างแล้ว:

- ประสบการณ์ AI มัลติ-โหมดที่ทำงานได้ทั้งในเบราว์เซอร์และบนเซิร์ฟเวอร์
- การรวมโมเดล AI ภายในเครื่องโดยใช้เฟรมเวิร์กและ SDK ของ Java สมัยใหม่
- บริการ Model Context Protocol แรกของคุณเพื่อดูว่าการผนวกเครื่องมือกับ AI ทำงานอย่างไร
